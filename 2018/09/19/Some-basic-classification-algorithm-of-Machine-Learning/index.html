Some basic classification algorithm of Machine Learning

Posted on 2018-09-19 | Edited on 2018-09-22 | In Machine Learning

Today lets review some basic classification algorithm of Machine Learning

As we know, Machine Learning generally divided to Supervised Learning and Unsupervised Learning and Supervised Learning has two branches, one is Classification, the other one is Regression. Today we just focus on Supervised Learning‘s Classification algorithms.

1. Decision Tree

First, lets take a quick look:

Basically, this algorithm make a classification by different decision node. Like above picture, we can judge Tim play or not play by some key attributes.

If Overcast, ok, he absolute plays. However, if Sunny or Rain, it is not very clearly. We should go to the next level to make a decision.

Obviously, the top node is the most important, the more deeper level, the less importance. If there are many features, how can we sort these features by importance?

The answer is Entropy.

According to Entropy, to define how much information one message contains, Uncertainty is the key value. The more uncertainty the more information we should get. And we can use bit as the unit to calculate one message’s entropy. Like this:
$$
H(x) = - \sum_{x_1}^{x_n}P\left(x\right)\log_2 P(x)
$$
Suppose we have five features, like A, B, C, D, E. So A’s Information Gain is :
$$
Gain(A) = Info(D) - Info{\over{A}}(D)
$$
Take an example to see the calculation details:

So, the Age’s information is:
$$
Gain(age) = Info(D) - Info_{\over{age}}(D) = 0.940 - 0.694 = 0.246bits
$$
And then, we can make a loop to calculate every feature’s information to sort them. The top node is the highest score, obviously.

In this algorithm, we should keep all features are discrete. Even a feature is continuous, we also need make a discretization.

Advantage:

  1. Intuitive
  2. Easy to understand.

Disadvantage:

  1. Only deal with discrete
  2. small-scale data
  3. Easy Overfit

3. Support Vector Machine (SVM)

Before Deep Learning showing up, SVM is the most successful and the best algorithm in the machine learning.

Suppose we trade a data point as a p-dimensional vector, the target of SVM is help us to find a (p - 1) - dimensional hyperplane which can separate such points, just like picture above. One reasonable choice as the best hyperplane is the one that represents the largest separation, or margin between the two classes. So we choose the hyperplane so that the distance from it to the nearest data point on each side is maximized. If such a hyperplane exists, it is known as the maximum-margin hyperplane and the linear classifier it defines is known as a maximum margin classifier or equivalently, the perceptron of optimal stability.

There are two kinds of SVM:

  1. Linear separable
  2. Linear inseparable or Nonlinear classification

1. Linear Separable

If the train dataset of n points like this:
$$
(\overrightarrow{x_1}, y_1),…, (\overrightarrow{x_n}, y_n)
$$
An hyperplane can be written as the set of points x satisfying
$$
\overrightarrow{w} * \overrightarrow{x} - b = 0
$$
W: weight vector
$$
W = {w_1, w_2,…, w_n}
$$

The points which locate on the hyperplane are called Support Vectors

Like point A and B, we call them are Support Vectors

The vertical distance between hyperplane and support line is:
$$
\frac{1}{\mid\mid w \mid\mid}
$$
So the Large Margin is:
$$
\frac{2}{\mid\mid w \mid\mid}
$$

2. Linear inseparable

Two steps:

  1. Transform points into higher dimensional spaces using nonlinear mapping
  2. Find a linear hyperplane in this higher dimensional space

Watch this video can make you understand more easier.

So, how to transform data points to the higher space?

If we have a 3 - dimensional vector:
$$
X = (x_1, x_2, x_3)
$$
it’s 6 - dimensional is this :
$$
(x_1, x_2, x_3, x_1^2, x_1x_2, x_1x_3)
$$
It’s called Dot product.

Obviously, the transform’s calculation is big, do we have a better switch?

Kernel trick

$$
K(x_i,x_j) = \phi(x_i) * \phi(x_j)
$$

There are some common Kernel functions:

  1. Polynomial kernel of degree h:
    $$
    K(x_i, x_j) = (x_i*x_j+1)^n
    $$

  2. Gaussian radial basis function kernel:

    $$
    K(x_i, x_j) = e^{-{\mid\mid x_i - x_j \mid\mid^2} / 2\sigma^2}
    $$

  3. Sigmoid function kernel:

    $$
    K(x_i, x_j) = \tan{(\kappa x_ix_j - \sigma)}
    $$

4. Neural Network

Firstly, it’s not actual neural, it just looks like neural because it has lots of nodes and connections. Before I learn this algorithm, I think this algorithm has some magic that can send signal like human neural. Actually, it just some loop to optimal parameters that can predict result. Of course, among these machine learning algorithms, it has the best accuracy.

NN is based on a collection of connected units or nodes called artificial neurons which loosely model the neurons in a biological brain. Each connection, like the synapses in a biological brain, can transmit a signal from one artificial neuron to another. An artificial neuron that receives a signal can process it and then signal additional artificial neurons connected to it.

The NN can be described as a multilayer feed-forward neural network and it use Backpropagation to optimize parameters.

There are three components in this network:

  1. Input layer
  2. Hidden layer
  3. Output layer

Every layer is formed by node. The begin layer aka Input layer accept train data as beginning. We sum previous layer’s nodes and multiply by weight which the edge between previous node and target node. Then add a bias and be transformed by a activation function to get final result which is current node’s value also is the next layer’s node’s input. Below picture can draw this process:

Normally, we can initial weights and bias randomly, between -1 and 1 or -0.5 ~ 0.5. Each node has its own bias.

This process just forward, so where is Backpropagation ?

Backpropagation

We can get a predicted value after arrival to the output layer, then compare with target value (true value). It’s not a perfect word, so it must have error between predicted value and target value, then we can reverse the calculation direction to update every edge’s weight and every node’s bias.

i means current layer

For the output layer (O means output, T means the true value):
$$
Err_j = O_j(1 - O_j)(T_j - O_j)
$$
For the hidden layer:
$$
Err_j = O_j(1 - O_j)\sum_k{Err_kW_{jk}}
$$
Weight update (l means learning rate, you can image the step of each update. When l is too big, we can not reach the right point because it’s easy to jump over. When l is too small, the efficiency is low):
$$
\Delta{w_{ij}} = (l)Err_jO_i
$$

$$
w_{ij} = w_{ij} + \Delta{w_{ij}}
$$

Bias update:
$$
\Delta\theta_j = (l)Err_j
$$

$$
\theta_j = \theta_j + \Delta\theta_j
$$

When to stop:

  1. Weight < Threshold
  2. Accuracy of predict < Threshold
  3. Loop >= Max
