Some basic classification algorithm of Machine Learning

Posted on 2018-09-19 | Edited on 2018-09-20 | In Machine Learning |

Today lets review some basic classification algorithm of Machine Learning

As we know, Machine Learning generally divided to Supervised Learning and Unsupervised Learning and Supervised Learning has two branches, one is Classification, the other one is Regression. Today we just focus on Supervised Learning‘s Classification algorithms.

1. Decision Tree

First, lets take a quick look:

Basically, this algorithm make a classification by different decision node. Like above picture, we can judge Tim play or not play by some key attributes.

If Overcast, ok, he absolute plays. However, if Sunny or Rain, it is not very clearly. We should go to the next level to make a decision.

Obviously, the top node is the most important, the more deeper level, the less importance. If there are many features, how can we sort these features by importance?

The answer is Entropy.

According to Entropy, to define how much information one message contains, Uncertainty is the key value. The more uncertainty the more information we should get. And we can use bit as the unit to calculate one message’s entropy. Like this:
$$
H(x) = - \sum_{x_1}^{x_n}P\left(x\right)\log_2 P(x)
$$
Suppose we have five features, like A, B, C, D, E. So A’s Information Gain is :
$$
Gain(A) = Info(D) - Info{\over{A}}(D)
$$
Take an example to see the calculation details:

So, the Age’s information is:
$$
Gain(age) = Info(D) - Info_{\over{age}}(D) = 0.940 - 0.694 = 0.246bits
$$
And then, we can make a loop to calculate every feature’s information to sort them. The top node is the highest score, obviously.

In this algorithm, we should keep all features are discrete. Even a feature is continuous, we also need make a discretization.

Advantage:

  1. Intuitive
  2. Easy to understand.

Disadvantage:

  1. Only deal with discrete
  2. small-scale data
  3. Easy Overfit

2. K - Nearest Neighbor (KNN)

KNN is also a classification algorithm, it belongs to instance-based learning and is a lazy learning.

Just show an example is enough:

Film Name Fights Kissing Type
California Man 3 104 Romance
He is not really into Dudes 2 100 Romance
Beautiful Woman 1 81 Romance
Kevin Longblade 101 10 Action
Robo Slayer 3000 99 5 Action
Amped II 98 2 Action

Suppose we have these films and two types.

Now we should judge one film which has 18 fights and 90 kissing belongs to what kind of film.

We can simply change these films into Cartesian Coordinate System, like this:

Point X Y Type
A 3 104 Romance
B 2 100 Romance
C 1 81 Romance
D 101 10 Action
E 99 5 Action
F 98 2 Action

Then there is one point G, (18, 90), which group it closes more?

According above picture, we can easily get what’s mean of KNN.
$$
E(x,y) = \sqrt{\sum_{i = 0}^{n}(x_i - y_i)^2}
$$
Use this function to calculate every two points’ distance. We choose K as a custom parameter, which means we will choose number of K‘s points which are the nearest and use majority-voting to decide the unknown data’s type.

Advantage:

  1. Easy understand
  2. Easy coding
  3. We can use K to import algorithm’s robustness

Disadvantage:

  1. Take lots of memory

  2. Big O is big

  3. If one type of data is big enough, it can easily affect the accuracy of judgement. Like this:
