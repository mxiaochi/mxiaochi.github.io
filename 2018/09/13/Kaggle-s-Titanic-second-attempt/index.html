Kaggle's Titanic second attempt

Posted on 2018-09-13 | Edited on 2018-09-20 | In DS

Since first submit my prediction, it’s past almost 2 weeks. During this two weeks, I have studied some ML models and data visualisation from Faster Data Science Education and then know more about the work flow about data analysis, here is my conclusion.

This is my first submission score:

It’s about 7700+th, not good. Then I learned some advance skills to improve my score, the most important article which I get improved is Titanic Top 4% with ensemble modeling and I sum up some good ideas to list.

Improvement

1. Outlier Drop

Some data they give us is too big or small which will affect the regression heavily resulting in a bad prediction model. So we should drop them and I use IQR to judge who is the wrong guy.

2. Join Train & Test Data to process

At first, I just use train data to train my model, however it has two obvious disadvantages, One is when I finish clear up my train data and produce a prediction model, I should repeat again to deal with my test data. And the other one is joined data can make imputation more precise, since you are using more observations to calculate median.

3. Use Heat Map and Factor Plot to show relation

Obviously, Heatmap is convenience for us to know about some relations among attributes.

Also, FactorPlot can show single attribute how to affect target:

4. Using median to fill missing value

The author is so talented that he analysis the relationship of SibSp, Parch, Pclass and Age and decided to use SibSP, Parch and Pclass in order to impute the missing ages.

He uses Factor Box to show these relations:

The strategy is fill Age with the median age of similar rows according to Pclass, Parch and SibSp. It’s very convenience and convince.

5. If distribute is skewed which could lead to overweight, we can use log function to reduce it

At first we can see the distribution about Fare is very skewed:

we can use Log function to reduce it:

6. How to deal with string columns

  • Use Capital

  • Get dummies

7. Add new attribute into data frame by joining exist attribute

The author create Family Size cause we can imagine that large families will have more difficulties to evacuate, looking for theirs sisters/brothers/parents during the evacuation.

The family size seems to play an important role, survival probability is worst for large families.

8. Choose ML model via cross_val_score

We can use cross_val_score to compare some ML model in order to choose the best one.

9. Model ensemble

I think Model ensemble is the most powerful tool as me know. It can let us choose serval models to deal with data together.

Conclusion

So, that’s what I have learned from Yassine Ghouzam, he gives me lots of inspiration. And it makes me improve my score a lot:
