1. Concept

In most of time, we can calculate the result directly. For example, we go for lunch cost $10/person, if we have 10 people, the result = 10 * 10, if we have 20 people, the result = 20 * 10. As the number of people growing, we can change the factor to multiply. However, suppose the number of people is huge and change quickly, if we still simply change the factor and multiple, it is a time consuming algorithm.

On the other hand, we can image that we have a table which records the cost of different number of people we have calculated. And then, if a new number appeared, we search the table to find close number. Such as we should calculate cost of 90001 person and now we search the table and find a cost of 90000 person, so we just use this result plus $10 get the final result.
